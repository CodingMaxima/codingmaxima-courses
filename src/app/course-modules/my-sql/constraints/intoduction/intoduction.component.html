Constraints

Introduction Animation Video
NOT NULL
  • The NOT NULL constraint enforces a column to NOT accept NULL values.
  • Each row of the column must contain value.
  • You can not insert/update row with null values for that column when column set to NOT NULL
  • You can apply NOT NUL to the multiple columns in the table.

    • EXAMPLE

  • CREATE TABLE Employee (
  •    EMP_ID int NOT NULL,
  •    NAME varchar(100) NOT NULL,
  •    EMAIL varchar(50) NOT NULL,
  •    Age int
  • );
    • UNIQUE
  • The UNIQUE constraint ensures that all values in a column are different from each other.
  • Which means no duplicate values are allowed.
  • You can apply UNIQUE to multiple columns in the table.

    • EXAMPLE

  • CREATE TABLE Employee (
  •    EMP_ID int NOT NULL UNIQUE,
  •    NAME varchar(100) NOT NULL,
  •    EMAIL varchar(50) UNIQUE,
  •    Age int
  • );

    • EXAMPLE - 2

  • CREATE TABLE Employee (
  •    EMP_ID int,
  •    NAME varchar(100) NOT NULL,
  •    EMAIL varchar(50),
  •    Age int
  •    UNIQUE (EMP_ID, EMAIL)
  • );
    • PRIMARY KEY
  • A combination of a NOT NULL and UNIQUE.
  • Only one PRIMARY KEY is allowed per table.
  • PRIMARY KEY can be added to single column or combination of multiple columns.

    • EXAMPLE

  • CREATE TABLE Employee (
  •    EMP_ID int PRIMARY KEY,
  •    NAME varchar(100) NOT NULL,
  •    EMAIL varchar(50) UNIQUE,
  •    Age int
  • );

    • EXAMPLE - 2

  • CREATE TABLE Employee (
  •    EMP_ID int,
  •    NAME varchar(100) NOT NULL,
  •    EMAIL varchar(50) NOT NULL UNIQUE,
  •    Age int
  •    PRIMARY KEY(EMP_ID)
  • );
    • FOREIGN KEY
  • A FOREIGN KEY is the one that is used to link two tables together via PRIMARY KEY.
  • CHECK
  • CHECK constraint ensures that the values in a column satisfies a specific condition or not.
  • You can not insert/update row without satisfies the specified condition of the column.
  • You can apply CHECK to multiple columns in the table.

    • EXAMPLE

  • CREATE TABLE Employee (
  •    EMP_ID int PRIMARY KEY,
  •    NAME varchar(100) NOT NULL,
  •    EMAIL varchar(50) UNIQUE,
  •    Age int CHECK (Age > 18)
  • );
    • DEFAULT
  • DEFAULT constraint provides the default value of the column.
  • If no value is provided to the column when you insert row then the default value will be added to the cell.

    • EXAMPLE

  • CREATE TABLE Employee (
  •    EMP_ID int PRIMARY KEY,
  •    NAME varchar(100) NOT NULL,
  •    EMAIL varchar(50) UNIQUE,
  •    Age int
  •    Joining_Date date DEFAULT GETDATE()
  • );
