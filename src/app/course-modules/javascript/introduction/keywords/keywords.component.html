Keywords

  • var: Declares a variable (in older versions of JavaScript; let and const are recommended in modern JavaScript).
  • let: Declares a block-scoped variable that can be reassigned.
  • const: Declares a block-scoped constant variable with a fixed value.
  • if: Conditional statement used to execute code if a specified condition is true.
  • else: Used in conjunction with an if statement to execute code if the condition is false.
  • switch: A conditional statement that tests a value against multiple possible case values and executes code based on the matching case.
  • for: A loop statement used to repeatedly execute a block of code for a specified number of times.
  • while: A loop statement that repeatedly executes a block of code as long as a specified condition is true.
  • do: A loop statement similar to while, but it always executes the block of code at least once before checking the condition.
  • break: Used to exit a loop or switch statement prematurely.
  • continue: Skips the current iteration of a loop and continues with the next iteration.
  • return: Exits a function and optionally returns a value to the caller.
  • function: Declares a reusable block of code that can be called with arguments.
  • class: Defines a blueprint for creating objects, introduced in ES6 (ECMAScript 2015).
  • new: Creates an instance of an object or invokes a constructor function.
  • this: Refers to the current object or context within a function or method.
  • null: Represents the intentional absence of any object value.
  • undefined: Represents a variable that has been declared but has no assigned value.
  • true: Boolean value representing a true condition.
  • false: Boolean value representing a false condition.
  • try: Starts a block of code to be tested for errors (used with catch and finally for error handling).
  • catch: Catches and handles exceptions thrown in a try block.
  • finally: Contains code that will be executed after a try or catch block, regardless of whether an exception occurred.
  • throw: Throws an exception that can be caught and handled with try and catch.
  • instanceof: Checks if an object is an instance of a particular class or constructor.
  • typeof: Returns a string indicating the data type of a variable or expression.
  • delete: Removes a property from an object.
  • in: Checks if an object has a specified property.
  • with: Used to extend the scope chain for a block of code, but it's considered a discouraged feature and is not recommended.
  • export: Used in modern JavaScript modules to specify which functions, objects, or values are accessible outside of the module.
  • import: Used in modern JavaScript modules to import functions, objects, or values from other modules.
  • default: Used in ES6 module exports to specify a default export from a module.
  • async: Declares that a function will return a promise and can use await inside the function to pause execution until the promise is resolved or rejected.
  • await: Used within an async function to pause execution until a promise is resolved or rejected.
  • class: Defines a class, which is a blueprint for creating objects with methods and properties.
  • extends: Inheritance keyword used to create a subclass that inherits properties and methods from a parent class.
  • super: Calls the constructor or methods of a parent class within a subclass.
  • constructor: A special method in a class that is called when an object is created from that class.
  • get: Defines a getter method to access the properties of an object.
  • set: Defines a setter method to modify the properties of an object.
  • static: Defines a static method or property that belongs to the class itself, not to instances of the class.
  • delete: Removes a property from an object or an element from an array.
  • instanceof: Checks if an object is an instance of a particular class or constructor function.
  • typeof: Returns a string indicating the data type of a variable or expression.
  • void: Used before an expression to evaluate it and return undefined.
  • with: Used to extend the scope chain for a block of code, but it's considered a discouraged feature and is not recommended.
  • NaN: Represents a special value that indicates a "Not-a-Number" result in numeric operations.
  • Infinity: Represents positive infinity, which is a numeric value that exceeds the maximum representable value.
  • -Infinity: Represents negative infinity, which is a numeric value that exceeds the minimum representable value.
  • for...of: Used to iterate over the values of an iterable object (e.g., arrays, strings, maps, sets) in a concise way.
  • for...in: Used to iterate over the properties of an object, including its prototype chain.
  • yield: Used in generator functions to yield a value to the caller without exiting the function.
  • arguments: An array-like object available inside functions that holds the arguments passed to the function.
  • arguments.callee: Refers to the currently executing function inside a function expression or anonymous function.
  • arguments.length: Indicates the number of arguments passed to a function.
  • eval: Evaluates a string of JavaScript code in the current scope, but it's considered risky and not recommended due to security concerns.
  • debugger: Pauses the execution of code and allows you to inspect the call stack, variables, and other program state using a debugging tool.
