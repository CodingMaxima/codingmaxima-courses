Data Types

Introduction Problems

In JavaScript there are two types of datatypes Primitive, Non-primitive(Reference) data types.

JavaScript is an dynamic language(no need to mention the type it will automatically takes the type dynamically by using the JavaScript engine).

Primitive Data Types

Primitive data types are basic data types that represent single values. They are immutable, means their values cannot be changed once they are assigned.

Data Types Description
String It will represents the sequence of characters. Strings are enclosed in single or double quotes.
Number It will represents the numeric values,including integers and floating-point numbers
Boolean It will represents the values either true or false. Used for logical operations and conditional statements.
Undefined It will gives the undefined values that means represents a variable that has been declared but hasn't been assigned a value.
Null It will gives the null values(no values) or objects.
Symbol (ES6 and later) It will represents a unique and immutable value, primarily used as object property keys. Symbols are created using the Symbol function.
Non-Primitive(Reference) Data Types

Reference / non-primitive data types are more complex and can hold multiple values. They are stored as references to memory locations.

Data Types Description
Object It will represents a collection of key-value pairs, where keys are strings (or Symbols) and values can be of any data type, including other objects.
Array It will represents the ordered collection of values, which can be of mixed data types. Arrays are indexed starting from 0.
Function functions are considered first-class citizens, which means they can be assigned to variables, passed as arguments to other functions, and returned from functions. Functions are reference data types because they reference executable code.
