Error handlings

Error handling in JavaScript is an essential part of writing robust and reliable code. Errors can occur for various reasons, such as incorrect input, network issues, or unexpected behavior, and it's important to handle these errors gracefully to prevent crashes and provide a better user experience.

Try...Catch Statements
  • try...catch statements are the most common way to handle errors in JavaScript. They allow you to wrap a block of code that might throw an exception (an error) in a 'try' block and then catch and handle that exception in a 'catch' block.

    • Example code:

    try {{ "{" }}

    // Code that might throw an error

    {{ "}" }} catch (error) {{ "{" }}

    // Handle the error

    {{ "{" }}


  • If an error occurs in the 'try' block, it will jump to the 'catch' block, and you can access the error object (typically named error but can be any valid variable name) to get information about the error, such as its message.
  • You can also use multiple 'catch' blocks to handle different types of errors.
    • Throwing Custom Errors
  • You can throw your custom errors using the 'throw' statement. This allows you to create meaningful error messages and handle specific situations in your code.

    • Example code:

    function divide(a, b) {{ "{" }}

      if (b === 0) {{ "{" }}
        throw new Error("Division by zero is not allowed");
      {{ "}" }}
      return a / b;

    {{ "}" }}


  • You can also create custom error types by extending the 'Error' class or other built-in error types like 'TypeError' or 'SyntaxError'.

    • Example code:

    class CustomError extends Error {{ "{" }} 

      constructor(message) {{ "{" }}
        super(message);
        this.name = "CustomError";
      {{ "}" }}

    {{ "}" }}

    Finally Block
  • You can use a 'finally' block after the 'try...catch' block to execute code that should run regardless of whether an error occurred or not. This is useful for cleanup operations, like closing a file or a network connection.

    • Example code:

    try {{ "{" }}

    // Code that might throw an error

    {{ "}" }} catch (error) {{ "{" }}

    // Handle the error

    {{ "}" }} finally {{ "{" }}

    // Code that runs regardless of errors

    {{ "}" }}
