Syntax Errors
  • These errors occurs because of some miskes done in syntax and occurs at compile time.
  • These errors was the programmers responsibility to correct them. Once the errors are corrected then the program will execute.
    • Run time errors
  • These errors will occurs at the execution time of the program becauseof end user input, memory problems, programming logic etc..
  • The exception handling concept was appicable only for the run time errors and not for the syntax errors.
    • Default Exception Handling
  • Every exception in python is an object. For every exception type the corresponding class will be available.
  • Whenever an exception occurs the PVM(Python Virtual Machine) will create the related exception object & will check for the handling that exception code.
  • If the handling code is available then it will be executed an the rest of the code will be executed normally.
  • If the code is not available then PVM will terminates the program abnormally and prints the related exception info at console.then the remaining program after the exception won't be executed.
  • To prevent this abnormal termination exceptions can be handled explicitly.By using try-except blocks.
    • customized exceptions
  • These are also known as user defined exceptions. We can create our own exceptions
  • In user defined exceptions we have to use raise Keyword for handling the exception .
