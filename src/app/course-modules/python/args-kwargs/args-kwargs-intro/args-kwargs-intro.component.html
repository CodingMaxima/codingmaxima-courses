args
  • These can be written as *args(single star) and also called as Non-keyword arguments
  • They allows to pass only non-keyword arguments to functions.
  • * symbol should have to be placed before the parameter name to pass variable length arguments.
  • The arguments are passed as tuple.
    • kwargs
  • These can be written as *args(double star) and also called as Keyword arguments
  • They allows to pass only keyword arguments to functions.
  • ** symbol should have to be placed before the parameter name for mentioning this type of argument.
  • The arguments are passed as a dictionary.
    • args Example kwargs Example args and kwargs Usage

    In this example, the log_message function accepts variable positional arguments (*args) and keyword arguments (**kwargs). The positional arguments are combined into a single message string using " ".join(str(arg) for arg in args). The keyword arguments provide additional flexibility, such as specifying the log level, timestamp, user ID, and custom metadata. These arguments are retrieved using kwargs.get() with default values.

    You can customize the implementation of the logging function according to your requirements. For example, instead of printing the log entry, you could write it to a log file, send it to a logging service, or perform other logging-related operations.

    Using *args and **kwargs in this scenario allows you to create a versatile logging function that can handle different log entries with varying levels, messages, and additional metadata in a flexible manner.
