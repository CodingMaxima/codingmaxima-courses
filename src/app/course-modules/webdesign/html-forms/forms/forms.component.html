Forms Introduction

Introduction
  • An HTML form is used to collect user input. The user input is most often sent to a server for processing.
  • The <form> element is a container for different types of input elements, such as: text fields, checkboxes, radio buttons, submit buttons, etc.
  • The forms elements may contain one or more of following elements
    • <button>
    • <label>
    • <select>
    • <textarea>
    • <input>

In HTML, a button is a user interface element that allows users to perform actions or submit forms on a web page.

The <button> element is used to create a clickable button.

Buttons can trigger JavaScript functions, submit forms, or navigate to different parts of a web application or website when clicked by a user.

Here's an example of how you can create a button in HTML:

<button type="button">Click me</button>

Explanation of the attributes used:

  • type="button": Specifies the type of the button. In this case, it's a standard button that doesn't have any specific behavior by default.

In HTML, <label> and <select> are two separate elements that serve different purposes in creating user interfaces and forms.

<label> Element:

  • The<lable> element is used to define a label for an <input> element.
  • It helps users understand what kind of information or input is expected in an associated input field.
  • It's beneficial for accessibility purposes as well, as it improves screen reader support and enables users to click on the label text to focus on the associated input field.
Example
<label for="username">Username</lable>
<input type="text" id="username" name="username">

<select> Element:

  • The<select> element is used to create a dropdown list of options, allowing users to select one or more items from the list.
Example
<label for="cars">Choose a car:</lable>
<select id="cars" name="cars>
    <option value="volvo">Volvo</option>
    <option value="audi">Audi</option>
    <option value="bmw">BMW</option>
    <option value="mercedes">Mercedes</option>
</select>

In HTML, the <textarea> element is used to create a multi-line text input field within a form.

It allows users to enter multiple lines of text, making it suitable for larger inputs like comments, messages, or any form of free-form text.

Example of how the <textarea> element is structured in HTML:

Example

    <textarea id="message" name="message" rows="4" cols="50">

    <textarea>

Explanation of attributes:

  • <id>: Specifies a unique identifier for the <textarea> element. This is used to associate the <lable> with the <textarea> through the for attribute in the <lable> tag.
  • name: Provides the name of the <textarea> element. When the form is submitted, this name-value pair will be sent to the server.
  • rows: Defines the visible number of rows of the textarea. It determines how many lines of text will be visible by default.
  • cols: Specifies the visible width of the textarea, measured in the number of characters across.

In HTML (Hypertext Markup Language), input types are attributes used within the <input> element to specify the type of data that a user can input through a form on a web page.

Different input types serve various purposes and allow users to enter specific kinds of information. Here are some common input types in HTML:

  1. Text: <input type="text">
    • Allows users to input a single line of plain text. This is commonly used for names, addresses, search bars, etc.
  2. Password: <input type="password">
    • Similar to the text input, but the entered characters are masked for security reasons (displayed as asterisks or dots). This is used for sensitive information like passwords.
  3. Number: <input type="number">
    • Accepts numerical input. You can set min, max, and step attributes to define the range and increments.
  4. Email: <input type="Email">
    • Validates input as an email address. It typically includes basic email validation and may show an error if the input doesn’t match the email format.
  5. URL: <input type="url">
    • Validates input as a URL (web address). Similar to email input, it checks if the entered text is in a proper URL format.
  6. Checkbox: <input type="checkbox">
    • Allows users to select one or more options from a list of choices. Each checkbox operates independently.
  7. Radio: <input type="radio">
    • Allows users to select a single option from a list. All radio buttons within the same group are mutually exclusive (i.e., selecting one deselects the others in the group).
  8. File: <input type="file">
    • Enables users to upload files from their device to the web server. This creates a file input field along with a "Browse" button.
  9. Date: <input type="date">
    • Provides a date picker to choose a date. The appearance and behavior of this picker can vary among different browsers.
  10. Time: <input type="time">
    • Allows users to input a time. Similar to the date input, the appearance can vary across browsers.
  11. Color: <input type="color">
    • Presents a color picker interface for selecting a color.
  12. Range: <input type="range">
    • Creates a slider control allowing users to choose a value within a specified range.
