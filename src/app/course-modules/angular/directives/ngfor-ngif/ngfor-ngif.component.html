Structural Directives

Directives in Angular are powerful and reusable components that allow you to extend the behavior of HTML elements or attributes. They are one of the core building blocks of Angular applications and play a crucial role in manipulating the DOM, adding behavior, and creating reusable functionality.

There are two types of directives in Angular:

1.Structural Directives:
Structural directives change the DOM layout by adding, removing, or manipulating elements based on certain conditions. They are typically used with a star (*) prefix in the template.

*ngIf: Conditionally adds or removes elements from the DOM based on a condition. 


<div *ngIf="showElement">This element is shown when showElement is true.</div>

*ngFor: Loops through a collection and repeats elements based on the number of items in the collection. For example: 


<ul>
    <li *ngFor="let item of items">{{'{'}}{{'{'}} item {{'}'}}{{'}'}}</li>
</ul>

*ngSwitch: Conditionally displays one element from multiple possible elements based on a switch expression. For example: 


<div [ngSwitch]="value">
    <p *ngSwitchCase="'first'">This is the first case.</p>
    <p *ngSwitchCase="'second'">This is the second case.</p>
    <p *ngSwitchDefault>Default case.</p>
</div>

2.Attribute Directives:
Attribute directives change the appearance or behavior of an element, component, or another directive. They are typically used as HTML attributes.

ngClass: Conditionally applies one or more CSS classes based on certain conditions. For example: 


<div [ngClass]="{{'{'}}{{'{'}} 'highlight': isHighlighted, 'error': hasError {{'}'}}{{'}'}}">Styled div element</div>

ngStyle: Conditionally applies CSS styles based on certain conditions. For example: 


<div [ngStyle]="{{'{'}}{{'{'}} 'color': textColor, 'font-size': fontSize {{'}'}}{{'}'}}">Styled div element</div>

ngModel: Provides two-way data binding for form elements, allowing you to bind form controls to component properties and update both the component and UI simultaneously. For example: 


<input type="text" [(ngModel)]="username">

Create a student registration form using Angular. The code provided consists of multiple components: `app.component`, `header.component`, and `student-form.component`, along with their respective HTML templates.

Here's a breakdown of the components and their HTML templates:

1.app.component:

  • Contains an `<app-student-form>` component, which renders the student registration form.

2.header.component:

  • Contains an `<h1>` element with the text "Student Form" as the header title.

3.student-form.component:

  • Includes an `<app-header>` component.
  • Contains a `<form>` element with the following fields:
    • Name: An `<input>` element with an `id` of "name" and a corresponding `[(ngModel)]` binding to update the name property when the user types.
    • Age: An `<input>` element with an `id` of "age" and a corresponding `[(ngModel)]` binding to update the age property when the user types.
    • Email: An `<input>` element with an `id` of "email" and a corresponding `[(ngModel)]` binding to update the email property when the user types.
  • Displays a `<button>` element with a `(click)` event binding to trigger the `onSubmit()` method when clicked.
  • Includes a `<table>` element that displays the entered student data:
    • The table has a header row with three columns: "Name," "Age," and "Email."
    • The table body displays a single row with the corresponding values from the `name`, `age`, and `email` properties.
  • The `app.component` should display the title variable as the `title` of the web page.
  • The `header.component` should display the text "Student Form" as an `<h1>` header.
  • The `student-form.component` should update the `name`, `age`, and `email` properties based on user input and display the entered data in the table.
  • The `onSubmit()` method should display an alert with the submitted form data.
