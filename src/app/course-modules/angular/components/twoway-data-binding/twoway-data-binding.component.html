Two Way Data Binding

Two-way data binding in Angular is a powerful feature that enables automatic synchronization of data between the component and the UI. It allows you to bind a component property to an input element in the template and have changes in either the component or the UI reflected in both directions.

Two-way data binding is denoted by the banana-in-a-box syntax ( `[( )]` ) and is a combination of property binding and event binding. It simplifies the process of keeping the component and UI in sync by automatically updating the property value when the UI changes, and vice versa.

Here's a more detailed explanation of two-way data binding in Angular:

Example-1: Binding to an input element:


<input [(ngModel)]="username">

In this example, the `[(ngModel)]` directive is used for two-way data binding. It binds the `username` property of the component to the value of the input field. Any changes made to the input field will update the `username` property, and changes to the `username` property will update the input field.

Example-2:Updating component and UI simultaneously:


<input [(ngModel)]="username">
<p>Welcome, {{'{'}}{{'{'}} username {{'}'}}{{'}'}}!</p>

Here, the value entered in the input field will be simultaneously displayed in the <p> element through interpolation. Any changes to the input field or the username property will be instantly reflected in both places.

Example-3: Handling different types of input elements:


<input type="text" [(ngModel)]="name">
<input type="checkbox" [(ngModel)]="isChecked">

Two-way data binding can be used with various types of input elements, such as text inputs, checkboxes, radio buttons, and selects. The `[(ngModel)]` directive handles the synchronization of data between the component and the UI for these elements.

Property binding supports not only string values but also other types such as numbers, booleans, arrays, and objects. It can also handle complex expressions and method calls.

It's important to note that two-way data binding requires the `FormsModule` to be imported and added to the `imports` array in your module. The `FormsModule` provides the necessary directives for two-way binding, including `ngModel`.

Two-way data binding is a powerful feature in Angular, simplifying the process of keeping the component and UI synchronized. However, it's important to use it judiciously and consider the performance implications, especially in complex applications with large data sets.

In scenarios where you only need to update the component or the UI independently, you can use one-way data binding techniques like property binding or event binding separately.
