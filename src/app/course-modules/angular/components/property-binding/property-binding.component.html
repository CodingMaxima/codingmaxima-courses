Property Binding

Property binding is a data binding technique in Angular that allows you to set the value of an HTML element's property or attribute based on the value of a component property. It establishes a unidirectional flow of data from the component to the template.

Property binding is denoted by square brackets (`[ ]`) and can be used to bind a component property to an HTML element's property, attribute, or a directive's input property.

Here's a more detailed explanation of property binding in Angular:

Example-1: Binding to an element's property:


<button [disabled]="isDisabled">Click me</button>

In this example, the `disabled` property of the `<button>` element is bound to the `isDisabled` property in the component. If `isDisabled` is true, the button will be disabled; otherwise, it will be enabled.

Example-2: Binding to an attribute:


<img [src]="imageUrl" alt="Product Image">

Here, the `src` attribute of the `<img>` element is bound to the `imageUrl` property in the component. The value of `imageUrl` will be dynamically inserted as the source of the image.

Example-3: Binding to a directive's input property:


<input [ngModel]="username">

This example demonstrates property binding with the `ngModel` directive, which is used for two-way data binding in Angular forms. The `ngModel` directive's input property is bound to the `username` property of the component. Any changes made to the `username` property will be reflected in the input field, and vice versa.

Property binding supports not only string values but also other types such as numbers, booleans, arrays, and objects. It can also handle complex expressions and method calls. 


<p [style.color]="isActive ? 'green' : 'red'">{{'{'}}{{'{'}} statusMessage() {{'}'}}{{'}'}}</p>

In this case, the `style.color` property of the `<p>` element is bound to an expression that determines the color based on the value of the isActive property. The `statusMessage()` method is also invoked to display a dynamic message within the paragraph element.

Remember, property binding is a one-way binding technique, meaning that changes to the component property will update the template, but changes in the template will not affect the component property. For two-way binding, you can use the combination of property binding and event binding using the banana-in-a-box syntax `[( )]` or utilize Angular's built-in two-way binding directives like `ngModel`.
