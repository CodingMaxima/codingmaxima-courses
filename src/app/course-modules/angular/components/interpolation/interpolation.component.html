Interpolation

Interpolation is a data binding technique in Angular that allows you to embed expressions inside the text content of HTML elements or attributes. It provides a way to dynamically display data in your Angular templates.

In Angular, interpolation is denoted by double curly braces (`{{'{'}}{{'{'}}{{'}'}}{{'}'}}`) and can be used in HTML elements, attributes, and event bindings. When the template is rendered, the expressions within the curly braces are evaluated, and their corresponding values are displayed in the UI.

Here are a few examples to illustrate how interpolation works:

Example-1: Displaying a variable value:


<h1>Welcome, {{'{'}}{{'{'}} username {{'}'}}{{'}'}}!</h1>

In this example, assuming you have a `username` property defined in your component class, the value of `username` will be dynamically inserted into the `<h1>` element. If `username` is "John", the rendered output will be: "Welcome, John!".

Example-2: Performing calculations and displaying results:


<p>The sum of {{'{'}}{{'{'}} num1 {{'}'}}{{'}'}} and {{'{'}}{{'{'}} num2 {{'}'}}{{'}'}} is {{'{'}}{{'{'}} num1 + num2 {{'}'}}{{'}'}}.</p>

In this case, assuming you have `num1` and `num2` properties in your component, their values will be added together and the result will be displayed within the paragraph element . For example, if `num1` is 5 and `num2` is 3, the rendered output will be: "The sum of 5 and 3 is 8."

Example-3: Accessing object properties:


<p>User details: {{'{'}}{{'{'}} user.name {{'}'}}{{'}'}}, {{'{'}}{{'{'}} user.age {{'}'}}{{'}'}}</p>

If you have an object named `user` with properties `name` and `age`, you can directly access them using interpolation. The values will be displayed within the paragraph element.

Interpolation is a simple and convenient way to display dynamic data in your Angular templates. It's important to note that interpolation is a one-way binding technique, meaning that changes to the component properties will update the template, but not vice versa. To achieve two-way binding, you would need to use other techniques such as property binding or two-way binding with ngModel.

We have to Remember to properly handle cases where the data being interpolated might be undefined or null to avoid errors. We can use the Angular's safe navigation operator ( `?.` ) or handle such scenarios in your component logic.
