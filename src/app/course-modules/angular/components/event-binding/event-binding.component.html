Event binding

Event binding is a data binding technique in Angular that allows you to respond to user interactions, such as button clicks, mouse movements, and keyboard events, by executing methods in your component class. It establishes a way to capture and handle events triggered by the user.

Event binding is denoted by parentheses (`( )`) and can be used to bind event handlers to HTML elements or directives.

Here's a more detailed explanation of event binding in Angular:

Example-1: Binding to a native HTML event:


<button (click)="handleClick()">Click me</button>

In this example, the `(click)` event of the `<button>` element is bound to the `handleClick()` method in the component. When the button is clicked, the `handleClick()` method is executed.


Example-2: Binding to a custom event:


<app-custom-component (customEvent)="handleCustomEvent($event)"></app-custom-component>

Here, a custom event named `customEvent` emitted by the `app-custom-component` is bound to the `handleCustomEvent()` method in the parent component. The `$event` object passed as an argument to the method contains additional data or information emitted by the child component.


Example-3: Binding to keyboard events:


<input (keydown)="handleKeyDown($event)">

This example binds the `(keydown)` event of the `<input>` element to the `handleKeyDown()` method in the component. The `$event` object provides information about the keyboard event, such as the key that was pressed.


Event binding allows you to create dynamic and interactive user experiences by responding to user actions. You can use event binding to handle user input, trigger actions, update component properties, and interact with services or APIs. 


<button (click)="counter = counter + 1">Increment</button>
<p>Current count: {{'{'}}{{'{'}} counter {{'}'}}{{'}'}}</p>

In this case, clicking the "Increment" button triggers the `(click)` event, which updates the `counter` property in the component by incrementing its value. The updated value is then displayed within the `<p>` element using interpolation.

You can also use event binding in combination with other techniques like property binding and two-way binding to create more advanced functionality.

Remember, event binding is a one-way binding technique, meaning that it allows you to handle events and update component properties, but changes in the component won't automatically update the template. For two-way binding, you can use the combination of event binding and property binding using the banana-in-a-box syntax `[( )]` or utilize Angular's built-in two-way binding directives like `ngModel`.

  • The `header.component` should display the text "Student Form" as an `<h1>` header.
