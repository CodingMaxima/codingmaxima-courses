Abstract Class (vs) Interface

introduction
Abstract Class Interface
Definition A class that cannot be instantiated and can contain both abstract and non-abstract methods. A blueprint of a class that defines a set of methods that a class must implement.
Instantiation Cannot be directly instantiated. Cannot be directly instantiated.
Inheritance Can provide implementation details and can be extended by a subclass using the "extends" keyword. Can be implemented by multiple classes using the "implements" keyword.
Method implementation Can have both abstract and non-abstract methods. Can only have abstract methods by default, but in Java 8 and later, it can also have default and static methods.
Fields Can have instance variables, static variables, and constants. Can have only constants (static final variables).
Multiple inheritance Does not support multiple inheritance. Supports multiple inheritance by implementing multiple interfaces.
Relationship Represents an "is-a" relationship. Represents a "has-a" or "can-do" relationship.
Usage Used when there is a common functionality among closely related classes. Used when unrelated classes need to implement common behavior.
Extension Can extend only one abstract class at a time. Can implement multiple interfaces simultaneously.
Accessibility Can have different access modifiers for methods and fields. Methods and fields areimplicitly public.

When to Use Abstract Class and Interface

Use Abstract Class When:

An abstract class should be used when:

  • You want to provide a common base implementation for a group of closely related classes.
  • You need to define instance variables or non-static fields that are shared among subclasses.
  • You want to enforce an "is-a" relationship, where the subclasses are specialized versions of the abstract class.
  • You anticipate evolving the abstract class over time, adding new methods or modifying existing ones, without breaking the existing subclasses.
  • You don't need to implement multiple inheritance, as Java does not allow a class to extend multiple abstract classes.

Use Interface When:

An interface should be used when:

  • You want to define a contract that multiple unrelated classes can implement to provide a common set of methods.
  • You need to achieve multiple inheritance by allowing a class to implement multiple interfaces.
  • You want to decouple implementation details from the interface definition.
  • You are working with a framework or library that expects objects to adhere to a specific interface.
