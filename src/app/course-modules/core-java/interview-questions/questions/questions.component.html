    Question-5

    What are Strings in Java?

  • A string can be created in three way in java i.e 
        String literal
        String Buffer 
        String Builder
  • When string literal is used to create a string, the data is stored in a string pool memory
  • When we modify the string literal it will create a new reference in string pool memory and assign that reference to the modified variable
  • This makes String as immutable
  • When a String is created using string buffer or String builder, the object can be modified any number of time
  • The main difference between String Buffer and String builder is String buffer is “Thread safe” and String builder is “Non Thread Safe”
  • If constant modifications are needed it is suggested to either string buffer or string builder because string literal wastes lots of string pool memory ( Answer this point when asked which is best method to use)

    What are Final, Finally and Finalize ?

    Final : Once a data is declared as final it cannot be modified further

  • When used before a variable the value cannot be changed
  • When used before a method the method cannot be overrided
  • When used before a class the class cannot be extended further

    • Finally : Finally is block used in exception handling which helps to execute critical operations even any exceptions occurred.

    Finalize: Finalize is a method used to perform cleanup operations on unmanaged.

  • Garbage collector implicitly performs the cleanup operation at the end of a program
  • If we need it in the middle of a program execution to free the memory we can use finalize method

    What is Polymorphism?

    Polymorphism is a mechanism which allows to perform a single task in different ways.

    There are two types of polymorphism

  • Compile time polymorphism
  • Runtime polymorphism

    • Compile-time polymorphism (Overloading): Compile time polymorphism is also known as Overloading, it states that a class can have multiple methods with same but with different signature i.e no of parameters and types of parameters. Each method stays as independed

    Run-time Polymorphism (Overriding): Run-time polymorphism is also known as Overriding, it states that a class and sub class can have exactly same methods. When the method is called it is decided at the run time to execute which class method based on the object.

    What is Exception?

    An exception is an abnormal situation which obstructs the actual flow of a program.

    Exceptions are classified of two types

  • Checked Exceptions
  • Unchecked Exceptions

    • Checked Exceptions: Checked exceptions are also known as Compile time exceptions.

  • For compile time exceptions the compiler will warn to handle the exceptions as there is a high possibility of getting it.
  • Ex : SQL Exception,IO Exception

    • Unchecked Exceptions: Unchecked exceptions are also known as Run-time exceptions.

  • For Run-time exceptions the compiler will not warn because the possibility of getting exception is very low.
  • Ex : Arithmetic Exception, Nullpointer Exception, Number format exception, IndexOutOfBoundException.

    • Exceptions can be handled by using :

    Try-Catch block:

  • The code that tends to throw exception will be written in the Try block
  • Catch block will catch the exception
  • A try block can have multiple catch blocks with child to parent hierarchy
  • Rest of the program will continue to work

    • Using Throws Keyword :

  • Throws keyword can be used at method level to throw the exception
  • If throws keyword is used the program will stop from execution after throwing exception

    Difference between Wait() and Seep()?

    Wait():

  • Wait() is used to make a thread go to waiting state. The thread will not go to running state until notify() or notifAll() method is called.
  • notify() is used to invoke one thread from waiting state. Ex: thread.notify()
  • notifyAll() is used to make all threads from waiting state. Ex : thread.notifyAll()

    • Sleep():

  • Sleep() method is used to make a thread go to waiting state for a particular period of time
  • After that period of time the thread will automatically goes to running state
  • Ex : thread.sleep(1000) - 1000 is milli seconds
