Introduction

Java Features JVM
Simple & secure
  • Simple: Java is easy to learn, understand, write, read & its syntaxes are simple, clean,easy for understanding purpuse.

  • Java programs are easy for creation and implemention while compared to other languages such as C, C++ because the complex and confusion concepts in C and C++ like explicit pointers& storage classes are not included in Java

  • Secure : In todays's generation security was the most important for every application. In java the secure features enable us to develop virus free, temper free system. Java program always runs in Java runtime environment with almost null interaction with system OS,so it is more secure.
    • Platform Independent
  • Java is an platform independent language means suports the WORA(Write Once And Run Anywhere) i.e., the programs written on one system can be easily executed on the other operating systems/ machine without any modifications.

  • Java source code is compiled using Java Compiler. The compiler converts the source code into an intermediate code called the byte code. The JVM (Java Virtual Machine) converts this byte code into the machine-dependent form. The JVM can execute byte code on any platform or operating system on which it is present.

  • Few examples of platform independent are Windows, Linux, Sun Solaris, Mac/OS, etc.
    • Object Oriented
  • Object Oriented means one can organize the software as a combination of different types of objects which has both data and behavior

  • Java is an fully Object Oriented programming language & programs are developed by using the Objects and classes

  • Oops has some basic important concepts such as Inheritance, Polymorphism, Abstraction, Encapsulation
    • High Performance
  • JIT(Just In Time compiler) enables high performance in java. It converts byte code into the machine code machine language and then JVM(Java Virtual Machine) starts execution.The execution speed is more comparitively to other languages.
    • Multi Threaded
  • A flow of execution is known as thread. JVM creates thread which is called main thread. User can create multiple threads by extending the thread class by implementing the runable interface. More than one task perform simultaneously.
    • Dynamic and extensable
  • During run time the program is in executable position by link up with some class libraries, objects and methods
    • Distributed
  • It has the facility to share single unit to different users & creating an network connection is easy by using this language
    • Robust
  • Robust means it can handle both the run-time errors and complie time errors as it checks the code during the compile and runtime. If any runtime error occurs is identified by the JVM it will not pass the program directly to the basic system. Instead, it will immediately terminate the program & stop's it from causing any harm to the system. Java has a strong memory management system & also supports the concepts of garbage collection and exception handling.
