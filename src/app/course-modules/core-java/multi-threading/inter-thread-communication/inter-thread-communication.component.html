wait()
  • The thread which is expecting is responsibe to call wait() then immediately enters into waiting state.
  • If a thread calls wait() on an object it immediately releases the lock of particular object and enters into waiting state
  • The thread will wake up from the waiting state only after notify(), notifyall() methods is called.

  • syntax: public void wait()
  • syntax: public void wait(long ms, int ns)

    • notify()
  • The thread which is responsible to perform updation. After performing the updation it is responsible to call notify() then waiting thread will get the notification and continue its execution with those updated items.
  • If a thread calls notify() on any object then it releases the lock of the object may not immediately
  • By using the notify() can gives the notification for only one waiting thread. If multiple threads are waiting only one thread will be notified and the remaining threads have to wait for further notification
  • Which thread will be notified we can expect it depends on thread schedular
  • syntax: public void notify()
    • notifyall()
  • We can use notifyall() to give the notification for all waiting threads of a particular object. Even though multiple threads notify but the execution will be performed one by one because threads required lock and only one lock is available.
  • syntax: public void wait()
    • suspend()
  • Whenever we are suspending the thread by using the suspend() of the thread class then immediately the thread will enter into the suspend state.
  • Syntax: public void suspend()
  • one can suspend the thread for a particular period of time like milli sec and also in nano sec
    • resume()
  • For getting out the thread from the suspend state to the runnable state have to use the resume() so that the thread will execute as usual.
  • Syntax: public void resume()
  • These suspend(), resume() is not recommended for the use because these methods are deprecated(disapproval).
    • stop()
  • We can use this method for stopping a thread to execute.But this method is also deprecated so not recommanded for use.
  • Syntax: public void stop()
