Sorting an Array with Bubble Sort

Consider an array of integers: [5, 2, 8, 1, 9]

  1. Bubble Sort starts by comparing the first two elements in the array (5 and 2). If they are in the wrong order, they are swapped.
    • Array: [2, 5, 8, 1, 9]
  2. The comparison continues with the next pair of elements (5 and 8). Since they are already in the correct order, no swap is needed.
    • Array: [2, 5, 8, 1, 9]
  3. The process continues until the largest element "9" "bubbles" up to the end of the array. Each pass compares adjacent elements and swaps them if necessary.
    • Pass 1: [2, 5, 8, 1, 9]
    • Pass 2: [2, 5, 1, 8, 9]
    • Pass 3: [2, 1, 5, 8, 9]
    • Pass 4: [1, 2, 5, 8, 9]
  4. The array is not fully sorted yet, so we repeat the process from the beginning until the array is sorted completely.
  5. The next pass would yield: [1, 2, 5, 8, 9]. Since no swaps are required, the algorithm recognizes that the array is sorted and terminates.

Bubble Sort has an average time complexity of O(n^2), where n is the number of elements in the array. It is not the most efficient sorting algorithm for large datasets but is simple to understand and implement.

There are several other sorting algorithms like Insertion Sort, Selection Sort, Merge Sort, Quick Sort, and many more, each with its own approach and performance characteristics. The choice of algorithm depends on the specific requirements of the application and the characteristics of the input data.
