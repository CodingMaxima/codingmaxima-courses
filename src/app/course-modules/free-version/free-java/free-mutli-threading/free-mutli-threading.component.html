Thread Life Cycle

Introduction
NEW/BORN
  • This is the state were new thread begins.The process remains in this condition until the program starts the thread.Now the thread is not considered alive.
    • RUNNABLE
  • The thread is in running state after the new thread is started.In this stage the thread is considered to executie its functions.
  • A thread can also return to this runnable state from waiting / sleeping state.
  • In this state the thread is considered alive.
    • RUNNING
  • The thread is in running state after it calls the run() method. Now the thread begins the execution.
    • WAITING / BLOCKED
  • In this state the thread is alive but it is not eligible to run.
  • The thread was not in runnable state but also it will return to runnable state after sometime Ex:- sleep, block, wait
    • DEAD / TERMINATE
  • If the two threads are waiting for each other forever then that type of infinite waiting is called DEAD LOCK
  • In this state once the run method is completed then it will be terminated. Now the thread will not be alive
    • start()
  • It acts like an heart of multithreading . Without executing the thread class start method there is no way of starting the new thread in java.
  • In the program whenever start() is called then a new thread will be created which is responsible for the execution of the run method.
    • run()
  • In this method a new thread won't be created and this method will be executed just like a normal method call by the main method.
    • yield()
  • It is used to pause the current executing thread to give the chance for the waiting threads of same priority
  • If there is no waiting thread /all waiting threads with low priority then the same thread will continue execution.
  • If multiple threads are waiting with same priority then which thread has to be executed will be depends on thread schedular.
    • join ()
  • It is used to join one thread with the end of currently running thread / if a thread is executing until the completion of that thread the another thread will goes to waiting state after the completion of that thread this thread will be executed in continution.
    • sleep()
  • It is the method where thread doesn't want to perform any operation for an particular period of time
