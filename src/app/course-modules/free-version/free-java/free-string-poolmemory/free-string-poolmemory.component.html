Example-1

String s1 = "Maxima";

String s2 = "Encode";

String s3 = "maima";

From the above examples s1, s2, s3 3 strings were created and they were stored in the string pool. The s1, s2 will execute properly in s3 there is already the value assigned already in s1 so instead of taking it as new object in dtring pool it returns the referencr of the pooled instance (s1).

Example-2

String str = new String("Coding Maxima");

In the above example whenever the string object is created using the new keyword It will check that the string literal is already present in the string pool or not. If yes, then only one string will be created in the string . If no, then it will be first created in the string pool memory then later in the heap memory so in total 2 string objects will be created.
