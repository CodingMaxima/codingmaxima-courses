Data Types in My-SQL

Introduction

Data types are describes the type of data with sizes and values that can be stored in the variable.

In Reality, There Are A Lot Of Different MYSQL Data Types

Data Type Description
INT It is used for integer values.Its range was -2147483648 to 2147483647.It allows the maximum characheters of 255 in length.
SMALLINT It is used for small integer values.Its range was -32768 to 32767.It allows the maximum characheters of 255 in length.
TINYINT It is used for very small integer values.Its range was -128 to 127.It allows the maximum characheters of 255 in length.
MEDIUMINT It is used for middle range integer values.Its range was -8388608 to 8388607.It allows the maximum characheters of 255 in length.
BIGINT It is used for high length integer values.Its range was -9223372036854775808 to 9223372036854775807.It allows the maximum characheters of 255 in length.
DECIMAL It was used for fixed point Number.It has the maximum value size is 65 & default vale is 10.The number of values after decimals are state with d parameter.its max value is 30 & default value is 0
FLOAT It is used for the floating-point values.It is determined by using p parameter.Its values is in between 0-24
DOUBLE It is also used for the floating-point values.It is determined by using p parameter.Its values is in between 25-53
BIT It is used for bit value types.The number of bits can be stated with size & its range is in between 1-64.The default size value is 1
Data Type Description
CHAR It has fixed length strings & contains numbers, letters and special characters.Its size range is in between 0-255. The default value is 1.
VARCHAR It has variable length strings & contains numbers, letters and special characters.Its size range is in between 0-65535.
BINARY Equal to CHAR(), but stores binary byte strings. The size parameter states the column length in bytes. Default is 1
VARBINARY Equal to VARCHAR(), but stores binary byte strings. The size parameter states the maximum column length in bytes.
BOLB It is used for BLOBs (Binary Large Objects). It can hold data up to 65,535 bytes.
TINYBOLB It is used for BLOBs (Binary Large Objects). It can hold data up to 255 bytes.
MEDIUMBOLB It is used for BLOBs (Binary Large Objects). It can hold data up to 16,777,215 bytes.
LONGBOLB It is used for BLOBs (Binary Large Objects). It can hold data up to 4,294,967,295 bytes.
TEXT Holds a string.It allows the maximum characheters of 255 in length.
TINYTEXT Holds a string.It allows the maximum characheters of 255 in length.
MEDIUMTEXT Holds a string.It allows the maximum characheters of 16,777,215 in length.
LONGTEXT Holds a string.It allows the maximum characheters of 4,294,967,295 in length.
ENUM It is used when a string object having only one value & it can be chosen from a list of possible values. It contains 65535 values in an ENUM list. If you insert a value that is not in the list, a blank value will be inserted.
Data Type Description
DATE It was used for only date & Format: YYYY-MM-DD. The range is from '1000-01-01' to '9999-12-31'
DATETIME It Was used for date and time combination & Format: YYYY-MM-DD hh:mm:ss. The range is from '1000-01-01 00:00:00' to '9999-12-31 23:59:59'. Adding DEFAULT and ON UPDATE in the column definition to get automatic initialization and it updates to the current date and time
TIMESTAMP TIMESTAMP values are stored as the number of seconds since the Unix epoch ('1970-01-01 00:00:00' UTC) & Format: YYYY-MM-DD hh:mm:ss. The range is is between '1970-01-01 00:00:01' UTC -'2038-01-09 03:14:07' UTC. For Automatic initialization and updating to the current date and time can be stated using DEFAULT CURRENT_TIMESTAMP and ON UPDATE CURRENT_TIMESTAMP in the column definition
TIME It is used only when we have to mention time & Format: hh:mm:ss. The range is in between '-838:59:59'-'838:59:59'
YEAR It was in four-digit format & Values allowed in four-digit format: 1901 to 2155, and 0000. MySQL 8.0 does not support year in two-digit format.
