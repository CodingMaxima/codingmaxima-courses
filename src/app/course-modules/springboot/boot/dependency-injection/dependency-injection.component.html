Dependency Injection

Intro Types Example

In a software application, classes often depend on other classes or services to perform various tasks. The traditional approach involves creating instances of dependent classes within the class that needs them. This can lead to tightly coupled code making it challenging to change or extend the application.

Dependency Injection (DI) is a design pattern that focuses on making a class's dependencies explicit and providing them from external sources rather than creating them within the class itself. The primary goal of dependency injection is to achieve separation of concerns, improve testability and promote modular and maintainable code.

In Java Spring Boot, Dependency Injection is achieved using the Inversion of Control (IoC) container, which is responsible for managing the lifecycle of objects and their dependencies.

Spring Boot provides several ways to perform dependency injection.Among them there are 3 types that are most commonly used. They are

1.Constructor Injection :

  • It involves providing the dependencies of a class through its constructor.
  • Dependencies are passed as parameters when creating an instance of a class.
  • Dependencies are explicitly defined and cannot be changed after instantiation.

2.Method Injection :

  • It involves providing the dependencies of a class through methods.
  • Dependencies are passed as parameters to specific methods.
  • Dependencies can be injected at different points in the objects lifecycle which enables flexibility.

3.Setter Injection :

  • It involves providing the dependencies of a class through setter methods.
  • Setters are defined in the class to allow external code to inject dependencies after the object has been created.
  • It allows for changing dependencies after the object has been instantiated.

BENEFITS

1.Decoupling: DI helps in reducing the coupling between classes, making the code more modular and easier to maintain.

2.Testability: By providing dependencies externally, it becomes easier to substitute real implementations with mock objects during testiny.

3.Flexibility: DI makes it easier to change or extend the behavior of a sytem by replacing or modifying individual components without effecting rest of codebase.
